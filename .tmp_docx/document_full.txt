轻量级语义结构构建技术设计方案
一、设计目标
本项目面向典型航空电子通信系统，针对现有建模方法适配性差、静态攻击链分析能力薄弱及适航条款难以对应的问题，提出构建轻量级建模语义结构，以支持静态资产关系表达、攻击路径推演与标准条款映射的基础建模需求。该结构强调要素简洁、语义清晰、人工可维护，适用于威胁建模初期的原型验证与可视化分析。
二、设计原则
聚焦关键要素：仅保留威胁建模中必要的核心语义（资产、关系、威胁点、攻击路径），明确语义边界，避免语义膨胀，满足静态建模所需。
适配静态分析场景：强调系统拓扑结构与路径可达性建模能力，支持图结构基础上的静态攻击路径推演与多跳攻击链表达。
支持标准条款映射：语义元素设计与DO-326A等适航安全标准术语保持一致，便于后续文本级对应和审查材料的生成。
结构可读与人工可维护：模型采用有向图结构组织，清晰呈现资产-关系-威胁三层静态结构，支持节点级手工增改和图形化人工审查。

该启发式评分机制的目的是帮助团队对不同攻击路径进行优先级排序，通过一个定量化的评分工具来实现，而非预测攻击的确切成功概率。评分公式为：
path_probability = entry_likelihood_value × attack_success_factor × source_weight
其中，entry_likelihood_value表示攻击路径进入的可能性，与威胁点的等级相关，分为High（0.7）、Medium（0.5）、Low（0.3）；
attack_success_factor表示威胁点的攻击复杂度，影响攻击的成功概率，分为Low（1.0）、Medium（0.7）、High（0.4）；
source_weight表示攻击源的风险权重，取决于攻击源类型，分别为内部（0.9）、外部（0.7）和第三方（0.5）。这些数值并非真实世界中的攻击成功率，而是帮助团队对不同攻击路径进行相对比较和排序，从而指导优先级。
启发式方法具有简单直观、快速得出结果的优势，不需要大量数据支持，同时具有较好的可解释性，每个因子和数值都能与现实中的场景直接对应。参数的灵活性也使得评分系统能够根据经验和具体场景进行调整，确保了评分的一致性，避免了各分析员基于直觉做出不同判断的风险。
为了确保评分系统的准确性和可操作性，设定了以下治理要点：首先，新增 expert_modifier（默认值为1.0，取值范围0.5~1.5）和 expert_adjustment_note字段，所有人工调整需在评审会议纪要中备案；其次，在评分完成后，使用 max 归一化计算 normalized_score = raw_score / max(raw_scores)，确保不同模型评分的可比性；当 normalized_score < 0.15 时，路径被标记为“低优先级”，默认不进入进一步缓解分析。最后，评分参数由威胁建模负责人维护，任何调整均需引用证据（如测试结果、专家意见或事件复盘），并通过变更单更新评分配置版本。
伪代码示意如下：
# 计算路径评分
score = entry_likelihood_value * attack_success_factor * source_weight
# 如果需要人工调整，乘以 expert_modifier
score *= expert_modifier  # 默认 1.0，人工调整在此乘以系数
# 归一化处理，确保不同模型评分可比
normalized_score = score / max(all_scores)
# 如果归一化评分低于阈值，标记为低优先级
if normalized_score < 0.15:
    mark_as_low_priority(path_id)
该机制的优势在于其轻量化与启发式方法的契合，使得在不需要大量数据的前提下能够快速得出结果，且具备较高的可解释性和灵活性。

五、实现形式与输出方式
建模平台：采用轻量图数据库（如 Neo4j）进行语义建模与可视化；
语义数据格式：支持 JSON / CSV 表达结构，便于跨平台处理与批量处理；
图形展示样式：资产类型通过颜色/图标区分，攻击路径用箭线表示跳跃链路，威胁点以图标或标签挂载，确保图形直观可审查；

攻击路径推演算法示例（具体以实现为准）：
路径推演算法基于图遍历技术（如Dijkstra算法）进行，主要考虑两个核心要素：路径的有效性和约束条件。
路径有效性：路径推演首先通过图遍历计算出从源资产到目标资产的所有可能路径，每条路径表示从一个资产到另一个资产的传播链条。在图中，每条连接（边）具有方向性，限制了信息的传播方向，确保数据流只能沿着指定的路径传播。其次，每条路径的信任级别（trust_level）和加密机制（security_mechanism）会影响路径的有效性。如果路径经过低信任级别的连接（如trust_level为Low），或者路径上存在加密机制（如SSL/TLS），则该路径可能会被排除，无法继续传播。只有通过高信任连接且不受加密机制限制的路径才会被视为有效。
约束条件：除了信任级别和加密机制外，还需要考虑路径的最大跳数（max_hops）。这一约束控制了路径的最大长度，确保推演过程中不会产生过长的路径，导致计算资源的浪费。在路径计算完成后，将根据这些约束条件筛选出符合条件的路径，避免无效的计算并提高效率。
最终，算法将输出满足所有约束条件的有效路径。这些路径的评估结果将帮助安全团队识别潜在的风险路径并制定相应的缓解策略。

场景介绍：
中央的信息系统处于“信息系统安保边界”之内，是核心保护目标。有两个内部接入设备，分别是IS-PMAT和IS-P-RFB，前者通过802.3/802.11接入信息系统，后者通过802.11接入信息系统。有四个外部连接系统，客舱系统通过802.3与信息系统连接、地面支持系统通过USB与信息系统对接、蜂窝基站通过蜂窝通信接口与信息系统交互、机场 Wi-Fi通过 802.11 与信息系统相连。
资产类：
这里id的命名规则为<系统类型+资产标识>，其中IS表示内部设备，IF表示接口，SYS表示系统。资产类型为Terminal、Interface、Link、Data， 对应到DO-326A 提到的典型资产范围（保护目标）Systems、Interfaces / Entry Points、Communication Links、Data / Functions。Criticality主要基于安全影响和攻击链作用（对应DO-356A 的 Threat Propagation 分析）
